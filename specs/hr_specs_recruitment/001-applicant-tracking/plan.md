# Implementation Plan: Applicant Tracking

**Branch**: `001-applicant-tracking` | **Date**: 2026-05-11 | **Spec**: `specs/hr_specs_recruitment/001-applicant-tracking/spec.md`
**Input**: Feature specification from `specs/hr_specs_recruitment/001-applicant-tracking/spec.md`

## Summary

Add applicant tracking to the HR recruitment module so recruiters can create applicants, move them through stages, coordinate interviews, track sources and attachments, warn on duplicate-email entries, and convert hired applicants into employees without re-entering core identity data. The implementation will follow Odoo conventions and reuse the ORM, chatter, attachments, and activity timeline for traceability.

## Technical Context

**Language/Version**: Python 3.11-compatible Odoo add-on
**Primary Dependencies**: Odoo ORM and mail/chatter APIs, PostgreSQL, `python-phonenumbers` for phone normalization, `ir.attachment`, calendar/activity models
**Storage**: PostgreSQL via the Odoo ORM
**Testing**: Odoo module tests and integration-style functional checks; pytest only if the repository already standardizes on it
**Target Platform**: Odoo server on Linux/Unix for deployment, Windows-supported development workspace
**Project Type**: Odoo add-on / module
**Performance Goals**: Keep create, stage update, duplicate check, and employee conversion flows responsive for individual recruiter actions
**Constraints**: Preserve Odoo recruitment semantics, keep duplicate-email detection as a warning rather than a hard failure, and only allow employee conversion from the hired state
**Scale/Scope**: Enterprise HR usage with moderate applicant volume per company and full audit/history retention
